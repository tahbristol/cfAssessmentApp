		 7. "https://api.twitter.com/1.1/tweets/search/30day/

          Twitter discontinued making its tweet counts publically available, at least easily. There are third-party modules and apps to do it, but it requires a premium
					search account. The standard tier doesn't allow for specifying for particular urls so far that I've seen.
			    This is the premium search tier of the twitter api. It returns total counts per time period and can be scoped to a particular url supplied as a query param.
					We have to apply for access to the premium tier. I'm in the process of applying for the sandbox version of this, which has a request limit of 30/min while the paid
					version is 60/min. The premium tier allows for more query params and a longer query string.

			8. https://api.twitter.com/1.1/search/tweets.json?q=http%3A%2F%2Fwww.congressweb.com%2Fawi%2F196%3Fframed_parent_url_id%3D3DB21594-FE8A-71E1-4EE5EA73D071F235&result_type=recent&count=100

				This is using the twitter standard search. Returns each tweet object containing timestamp, tweet_id, user..etc. 100 is the limit of returns so the last 100 tweets.
				From Jan 26th to Jan 31 this search returns  ~100 tweet events. This is at least. AddThis returns 103 shares via twitter for the past week for this same url. Seems
				like the premium search is the way to go. It gives a 30 day history, gives counts daily, hourly and by the minute. The search query is double the length.
